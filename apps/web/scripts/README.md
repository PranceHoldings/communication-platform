# Development Scripts

このディレクトリには、開発サーバーの管理を簡単にするためのスクリプトが含まれています。

## 📋 利用可能なスクリプト

### 1. `dev-clean.sh` - クリーンな開発環境起動

**最も推奨される方法**。既存サーバーを停止→ビルド→開発サーバー起動。

```bash
pnpm run dev:clean
```

**実行内容:**

1. 既存のNext.jsプロセスを停止（ポート3000, 3001, 3002）
2. `.next` ディレクトリをクリーンアップ
3. プロジェクトをビルド
4. 開発サーバーをポート3000で起動

**使用タイミング:**

- セッション開始時
- ビルドエラーが発生した場合
- ポートコンフリクトが発生した場合
- クリーンな状態から始めたい場合

---

### 2. `dev-restart.sh` - クイック再起動

ビルドなしで既存サーバーを停止して開発サーバーを再起動。

```bash
pnpm run dev:restart
```

**実行内容:**

1. 既存のNext.jsプロセスを停止
2. 開発サーバーをポート3000で起動（ビルドなし）

**使用タイミング:**

- サーバーが応答しなくなった場合
- ポートだけクリアして再起動したい場合
- ビルドは不要だが再起動が必要な場合

---

### 3. `start-clean.sh` - プロダクションビルド起動

本番環境と同じビルドでサーバーを起動。

```bash
pnpm run start:clean
```

**実行内容:**

1. 既存のNext.jsプロセスを停止
2. `.next` ディレクトリをクリーンアップ
3. プロダクションビルドを作成
4. プロダクションサーバーをポート3000で起動

**使用タイミング:**

- デプロイ前のテスト
- 本番環境と同じ動作を確認したい場合
- パフォーマンステスト

---

### 4. `kill-port.sh` - 特定ポートのプロセスを停止

指定したポートで動作しているプロセスを強制終了。

```bash
pnpm run kill-port 3000
# または
bash scripts/kill-port.sh 3000
```

**使用タイミング:**

- 特定のポートだけクリアしたい場合
- "Port already in use" エラーが出た場合

---

## 🚀 推奨ワークフロー

### 毎日の開発開始時

```bash
cd apps/web
pnpm run dev:clean
```

### コード変更後の再起動

通常はホットリロードが動作しますが、必要な場合:

```bash
# 軽量な再起動（ビルドなし）
pnpm run dev:restart

# 完全な再起動（ビルドあり）
pnpm run dev:clean
```

### デプロイ前の確認

```bash
pnpm run start:clean
```

---

## 🔍 トラブルシューティング

### ポートが使用中のエラー

```bash
pnpm run kill-port 3000
pnpm run dev
```

### ビルドエラー

```bash
pnpm run clean
pnpm run dev:clean
```

### サーバーが応答しない

```bash
pnpm run dev:restart
```

---

## 📝 注意事項

- すべてのスクリプトは自動的にポート3000を使用します
- 既存のプロセスは自動的に停止されます
- `dev:clean` と `start:clean` はビルドを実行するため、数秒～数十秒かかります
- 開発中は基本的に `dev:clean` を使用することを推奨します

---

## 🛠️ カスタマイズ

他のポートを使用したい場合:

```bash
PORT=3001 pnpm run dev
PORT=3001 pnpm run start
```

または、`.env.local` に追加:

```env
PORT=3001
```
