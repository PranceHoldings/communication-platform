# AWS統合化分析レポート

**作成日**: 2026-03-04
**目的**: 品質を維持しながら、可能な限りAWSサービスに統合する

---

## 方針

1. **品質が最優先** - 品質低下は許容しない
2. **品質同等ならAWS優先** - 開発効率、コスト、統合性の向上
3. **専門サービスは維持** - AWSに代替がない場合は最適なサービスを選択

---

| -------------------------- | -------------- | --------------- | ----------------------- |
| **品質・自然さ**           |                |                 |                         |

| イントネーション           | ◎              | ○               | **ElevenLabs 優位**     |

| **機能**                   |                |                 |                         |

| 料金                       | $0.30/1000文字 | $0.004/1000文字 | **AWS 大幅優位**        |
| 無料枠                     | 10,000文字/月  | 500万文字/月    | **AWS 大幅優位**        |
|                            |                |                 |                         |
| **開発体験**               |                |                 |                         |
| セットアップ               | 簡単           | 簡単            | **同等**                |
| ドキュメント               | ◎              | ◎               | **同等**                |

**総合評価**:

- ✅ **品質**: ElevenLabs が圧倒的に優位（自然さ、感情表現）
- ✅ **機能**: 音声クローニングはElevenLabsのみ（重要機能）
- ✅ **AWS統合**: Polly が優位
- ✅ **コスト**: Polly が大幅に優位（75倍安い）

**推奨**:

- **プライマリ: ElevenLabs、フォールバック: Amazon Polly**
- 理由:
  1. **品質差が非常に大きい** - ユーザー体験に直結
  2. 音声クローニングは差別化機能
  3. コストは高いが、Alpha版では許容範囲（月$30-50程度）
  4. Pollyをフォールバックとして実装（AWS統合の利点を活用）
  5. Phase 2以降、シナリオに応じて使い分け可能

**実装方針**:

```typescript
// マルチプロバイダ実装
interface TTSProvider {
  synthesize(text: string, voiceId: string): Promise<AudioStream>;
}

class ElevenLabsProvider implements TTSProvider { ... }
class PollyProvider implements TTSProvider { ... }

// プライマリ: ElevenLabs、エラー時: Polly
const tts = new TTSService({
  primary: new ElevenLabsProvider(),
  fallback: new PollyProvider()
});
```

### 4. 3Dアバター生成

#### 現状: Ready Player Me

#### AWS代替: なし

**分析**:

- AWSには3Dアバター生成の専門サービスなし
- 代替案:
  1. AWS上に独自実装（コスト・期間が膨大）
  2. 他の3Dアバターサービス（Didimo, Wolf3D等）

**推奨**:

- **Ready Player Me 継続**
- 理由:
  1. 業界標準、品質・機能が優れている
  2. Photo Capture API（写真→3Dアバター）が有用
  3. AWS統合は不可だが、REST API経由で問題なし
  4. 代替実装のコストが高すぎる

## 変更まとめ

### 即座に変更（Alpha版）

1. **STT**: Azure Speech Services **維持** → 品質優先
2. **感情解析**: Azure Face API → **AWS Rekognition に変更**
3. **TTS**: ElevenLabs **維持** + AWS Polly フォールバック追加

### Phase 2で検討

1. **STT**: Azure → AWS Transcribe 移行検討（品質検証後）

## AWS統合度

### 変更前

- AWS: 50%（Bedrock, Lambda, Aurora, S3等）
- Azure: 25%（Speech, Face API）
- その他: 25%（ElevenLabs, Ready Player Me）

### 変更後（Alpha版）

- AWS: 70%（Bedrock, Rekognition, Polly, Lambda, Aurora, S3等）
- Azure: 10%（Speech のみ）
- その他: 20%（ElevenLabs, Ready Player Me）

### 目標（Phase 2）

- AWS: 80%（+ Transcribe）
- その他: 20%（ElevenLabs, Ready Player Me）
