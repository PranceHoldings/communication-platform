# リリースプラン

実装容易度、ユーザー価値、技術的リスクを考慮した段階的リリース計画。

## リリース戦略の原則

### 段階的価値提供

```
Alpha → Beta → v1.0 → v1.x → v2.0
  ↓       ↓      ↓      ↓      ↓
内部    限定   一般   機能   完全
テスト  公開   公開   追加   版
```

### 各リリース段階の定義

| 段階      | 対象ユーザー        | 品質基準                 | 機能範囲             |
| --------- | ------------------- | ------------------------ | -------------------- |
| **Alpha** | 開発チーム内        | 動作確認                 | コア機能のみ         |
| **Beta**  | 10-50組織（招待制） | 基本品質保証             | MVP機能              |
| **v1.0**  | 一般公開            | プロダクション品質       | SaaS基本機能         |
| **v1.x**  | 全ユーザー          | 高品質・高可用性         | エンタープライズ対応 |
| **v2.0**  | 全ユーザー          | エンタープライズグレード | 完全機能セット       |

---

## Alpha版（内部テスト）- 2ヶ月

**リリース目標日**: Phase 1完了時（開発開始から2ヶ月後）

**目標**: 基本的な会話セッションが動作することを検証

### 提供機能

#### ✅ コア機能（実装容易度: ⭐〜⭐⭐）

```
ユーザーストーリー:
「管理者として、プリセットアバターを選択して、
 基本的な面接シナリオで会話セッションを実行し、
 録画を確認できる」
```

**含まれる機能**:

1. **認証・組織管理** ⭐
   - メールアドレス + パスワード認証
   - 組織作成（1組織のみ）
   - ユーザー登録（管理者 + 一般ユーザー）

2. **アバター** ⭐
   - プリセット3Dアバター（3種類）
   - リアルタイムリップシンク
   - 基本表情（neutral, happy, confused）

3. **音声** ⭐
   - ElevenLabs TTS（英語・日本語各1音声）
   - Azure STTリアルタイム認識
   - マイク入力・音声ストリーミング

4. **シナリオ** ⭐⭐
   - プリセットシナリオ（面接練習 1種類）
   - 固定システムプロンプト（変更不可）
   - Claude API統合

5. **セッション実行** ⭐⭐
   - リアルタイム会話（WebSocket）
   - ブラウザ録画（ユーザー + アバター）
   - S3アップロード

6. **録画再生** ⭐
   - 基本的な動画プレイヤー
   - トランスクリプト表示（テキストのみ）

### 技術的マイルストーン

- [ ] Cognitoユーザー認証フロー動作
- [ ] Aurora Serverless v2接続確立
- [ ] Lambda API Gateway統合
- [ ] IoT Core WebSocket通信確立
- [ ] ElevenLabs TTS → ブラウザ音声再生
- [ ] Azure STT → リアルタイム字幕表示
- [ ] Claude API → アバター応答生成
- [ ] MediaRecorder → S3録画保存

### Alpha版の制約

❌ **含まれない機能**:

- カスタムアバター作成
- カスタムシナリオ作成
- 感情・音声解析
- レポート生成
- ベンチマーク機能
- 外部API連携
- ATS連携
- 多言語UI（日本語のみ）

### 成功基準

- [ ] 10セッション以上の連続実行が安定動作
- [ ] WebSocket切断率 < 5%
- [ ] 録画ファイル生成成功率 > 95%
- [ ] P95レスポンスタイム < 2秒（API）
- [ ] クリティカルバグ 0件

### リリース判断基準

**Go判断（Beta移行）**:

- すべての成功基準を満たす
- 開発チーム全員が「友人に勧められる」と判断

**No Go判断（追加修正）**:

- 成功基準を1つでも満たさない
- セキュリティ脆弱性の発見

## Beta版（限定公開）- 1.5ヶ月

**リリース目標日**: Phase 2完了時（開発開始から3.5ヶ月後）

**目標**: MVPとしてのプロダクトマーケットフィットを検証

### 提供機能

#### ✅ Alpha版機能 + 以下を追加

**1. アバター拡充** ⭐⭐

- プリセット3Dアバター（20種類）
- プリセット2Dアバター（10種類）
- アバター選択UI（カテゴリフィルタ）
- カスタムアバター作成（画像アップロード → 3D/2D生成）

**2. 音声拡充** ⭐⭐

- 音声プリセット（英語5種、日本語5種）
- 音声クローニング（ファイルアップロード）
- ブラウザ録音 → 音声クローニング

**3. シナリオ拡充** ⭐⭐

- プリセットシナリオ（5種類: 面接、語学、カスタマーサービス等）
- シナリオビルダーUI（簡易版）
- ユーザーが独自シナリオ作成可能
- シナリオ共有（組織内）

**4. AI管理（管理者）** ⭐⭐⭐

- AIプロンプトテンプレート管理UI
- プロンプト編集・テスト機能
- 変数システム（基本）
- プロンプトバージョン管理

**5. AIプロバイダ管理（管理者）** ⭐⭐⭐

- プロバイダ設定UI（Claude, GPT-4）
- プロバイダ切り替え機能
- 使用量トラッキング
- コストダッシュボード（基本）

### Beta版対象ユーザー

**想定**: 10-50組織（招待制）

**ターゲット顧客**:

- 採用担当者（企業HR部門）
- 語学学校・教育機関
- 研修企業

**募集方法**:

- ランディングページからの申し込み
- LinkedIn・Twitterでの告知
- 既存ネットワークからの紹介

**提供条件**:

- 無料（6ヶ月間）
- 月間セッション数制限: 50セッション/組織
- フィードバック提供義務
- NDA締結

### Beta版の制約

❌ **含まれない機能**:

- 感情・音声解析
- AIレポート自動生成
- ベンチマーク機能
- 外部API連携
- ATS連携
- 課金機能
- SSO/SAML
- 多言語UI（日本語・英語のみ）

### 成功基準

**定量指標**:

- [ ] 30組織以上の登録
- [ ] 月間アクティブ率 > 60%（登録組織のうち月1回以上利用）
- [ ] セッション完了率 > 80%（開始したセッションのうち完了）
- [ ] 週次リテンション（W1） > 40%
- [ ] NPS（Net Promoter Score） > 30

**定性指標**:

- [ ] 顧客インタビュー 20件以上実施
- [ ] 「代替手段がない」と回答 > 50%
- [ ] 有料化した場合「購入する」 > 30%

**技術指標**:

- [ ] システム稼働率 > 99%
- [ ] P95 APIレスポンスタイム < 500ms
- [ ] エラー率 < 1%

### リリース判断基準

**Go判断（v1.0移行）**:

- 定量指標のうち4つ以上達成
- 顧客インタビューで明確な課題解決を確認
- 有料化意向 > 30%
- P0/P1バグ 0件

**Pivot判断（方向転換検討）**:

- 月間アクティブ率 < 30%
- セッション完了率 < 50%
- 顧客インタビューで「不要」との声多数

## v1.0（一般公開）- 2ヶ月

**リリース目標日**: Phase 3完了時（開発開始から5.5ヶ月後）

**目標**: プロダクションレディなSaaSとして一般公開

### 提供機能

#### ✅ Beta版機能 + 以下を追加

**1. 解析・レポート** ⭐⭐⭐

- 感情解析（Azure Face API）
- 音声特徴解析（Azure Speech Analytics）
- AIレポート自動生成（Claude API）
- レポートPDFエクスポート
- 感情タイムライングラフ
- ハイライトシーン自動抽出

**2. トランスクリプト同期プレイヤー** ⭐⭐

- クリッカブルトランスクリプト
- 動画 ↔ テキスト双方向同期
- ハイライト表示（重要発言・改善点）
- トランスクリプト検索

**3. プラン・課金** ⭐⭐⭐

- Freeプラン（5セッション/月）
- Proプラン（$99/月、50セッション/月）
- Enterpriseプラン（カスタム見積もり）
- 使用量制限・クォータ管理
- 請求書払い（Enterpriseのみ、初期）
- ※Stripe統合は v1.1で実装

**4. マルチテナント完全対応** ⭐⭐⭐

- 組織間データ完全分離（Row Level Security）
- 組織設定・ブランディング
- 組織専用アバター管理
- ユーザー招待・ロール管理

**5. 管理者ダッシュボード** ⭐⭐

- 組織内セッション一覧・分析
- ユーザー管理UI
- 使用状況・コスト監視
- プロンプト・プロバイダ管理

### v1.0のポジショニング

**市場メッセージ**:

```
「AIアバターとの面接練習で、
 客観的フィードバックを即座に取得」

対象: 就職活動中の学生・転職希望者、企業HR部門
差別化: リアルタイムAI会話 + 自動解析レポート
```

**価格戦略**:

- Free: 個人ユーザー獲得（バイラルループ）
- Pro: 小規模企業・教育機関（メインターゲット）
- Enterprise: 大企業（高付加価値、カスタマイズ）

### v1.0の制約

❌ **含まれない機能**:

- プロファイルベンチマーク
- 外部API連携
- ATS連携
- プラグインシステム
- SSO/SAML（Enterpriseで手動設定のみ）
- カスタムレポートテンプレート
- 多言語対応（日本語・英語のみ）

### 成功基準

**ローンチ後1ヶ月**:

**獲得指標**:

- [ ] 登録組織数 > 200
- [ ] 有料プラン契約 > 20組織
- [ ] MRR（月次経常収益） > $5,000

**エンゲージメント指標**:

- [ ] DAU/MAU > 0.3（アクティブユーザー定着率）
- [ ] 月次リテンション（M1） > 50%
- [ ] セッション完了率 > 85%

**品質指標**:

- [ ] システム稼働率 > 99.5%
- [ ] サポートチケット対応時間 < 24時間
- [ ] CSAT（顧客満足度） > 4.0/5.0

**経済指標**:

- [ ] CAC（顧客獲得コスト） < $200
- [ ] LTV/CAC > 3
- [ ] チャーンレート < 10%/月

### リリース判断基準

**Go判断（v1.x移行）**:

- ローンチ後1ヶ月で獲得指標達成
- 有料プラン契約 > 20組織
- P0/P1バグ解決済み
- セキュリティ監査クリア

**Delay判断（リリース延期）**:

- セキュリティ脆弱性（高・重大）の発見
- データ損失リスクのある不具合
- 法務・コンプライアンス問題

**リリース目標日**: Phase 4完了時（開発開始から9.5ヶ月後）

**目標**: エンタープライズ顧客獲得と機能拡充

**ビジネス価値**:

**ターゲット顧客**:

- 教育機関（学生の成長可視化）

**実装容易度**: ⭐⭐⭐（中〜高）

**提供機能**:

- APIキー発行・管理（クライアント管理者）

- 階層的レート制限
- API使用状況ダッシュボード

**ビジネス価値**:

**ターゲット顧客**:

**提供機能**:

**ターゲット顧客**:

- 大企業（IT部門要件）

**実装容易度**: ⭐⭐⭐（中）

## リスク管理

### 主要リスクと対策

| リスク                 | 影響度 | 発生確率 | 対策                                             |
| ---------------------- | ------ | -------- | ------------------------------------------------ |
| **AI APIコスト超過**   | 高     | 中       | 使用量アラート、プロバイダ切り替え準備           |
| **WebSocket安定性**    | 高     | 中       | 自動再接続、フォールバック処理                   |
| **TTS/STT精度不足**    | 中     | 中       | マルチプロバイダ対応、ユーザーフィードバック収集 |
| **スケーラビリティ**   | 高     | 低       | サーバーレスアーキテクチャ、負荷テスト           |
| **データプライバシー** | 高     | 低       | 暗号化、GDPR対応、監査ログ                       |
| **競合参入**           | 中     | 中       | 差別化機能（ベンチマーク、ATS連携）優先          |
| **PMF未達成**          | 高     | 中       | Beta版で早期検証、Pivot準備                      |

### 緊急時対応

**データ損失**:

- RTO: 1時間以内
- RPO: 15分以内
- 対策: 自動バックアップ（15分間隔）、マルチAZ構成

**セキュリティ侵害**:

- 対応時間: 即座（24時間体制）
- 対策: AWS GuardDuty、侵害検知アラート、インシデント対応手順書

**システム障害**:

- 対応時間: 30分以内
- 対策: 自動フェイルオーバー、ロールバック手順、ステータスページ
