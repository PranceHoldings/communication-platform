# 多言語対応システム（i18n）

**最終更新:** 2026-03-05
**ステータス:** ✅ 実装完了

---

## 概要

Pranceプラットフォームはカスタム多言語対応システムを使用しています。
- **軽量・シンプル**: next-intlを使用せず、カスタム`useI18n`フックで実装
- **ネストされた翻訳**: JSON階層構造でキーを管理
- **型安全**: TypeScriptで型推論をサポート
- **ホットリロード**: 開発時に言語リソース変更が即座に反映

## アーキテクチャ

### 言語リソースファイル

```
apps/web/messages/
├── en/                    # 英語（デフォルト）
│   ├── common.json        - 共通文言
│   ├── auth.json          - 認証画面
│   ├── dashboard.json     - ダッシュボード
│   └── sessions.json      - セッション関連
└── ja/                    # 日本語
    ├── common.json
    ├── auth.json
    ├── dashboard.json
    └── sessions.json
```

### コアファイル

```
apps/web/lib/i18n/
├── provider.tsx           # I18nProvider, useI18n フック
├── messages.ts            # 言語リソース読み込み
└── middleware.ts          # 言語検出（Cookie > Accept-Language > デフォルト）
```

## 使い方

### 1. 翻訳を使用する（コンポーネント）

```tsx
'use client';

import { useI18n } from '@/lib/i18n/provider';

export default function MyComponent() {
  const { t, locale, setLocale } = useI18n();

  return (
    <div>
      {/* 基本的な翻訳 */}
      <h1>{t('common.app_name')}</h1>

      {/* パラメータ付き翻訳 */}
      <p>{t('dashboard.welcome', { name: 'John' })}</p>

      {/* ネストされたキー */}
      <p>{t('sessions.new.scenario.title')}</p>

      {/* 現在の言語 */}
      <p>Current locale: {locale}</p>

      {/* 言語切り替え */}
      <button onClick={() => setLocale('ja')}>日本語</button>
      <button onClick={() => setLocale('en')}>English</button>
    </div>
  );
}
```

### 2. 新しい翻訳キーを追加

**ステップ1: 英語リソースを追加**
```json
// apps/web/messages/en/dashboard.json
{
  "new_key": "Hello World",
  "nested": {
    "key": "Nested translation"
  },
  "with_param": "Hello, {name}!"
}
```

**ステップ2: 日本語リソースを追加**
```json
// apps/web/messages/ja/dashboard.json
{
  "new_key": "こんにちは世界",
  "nested": {
    "key": "ネストされた翻訳"
  },
  "with_param": "こんにちは、{name}さん！"
}
```

**ステップ3: コンポーネントで使用**
```tsx
const { t } = useI18n();
console.log(t('dashboard.new_key'));           // "Hello World" または "こんにちは世界"
console.log(t('dashboard.nested.key'));        // "Nested translation" または "ネストされた翻訳"
console.log(t('dashboard.with_param', { name: 'John' })); // "Hello, John!" または "こんにちは、Johnさん！"
```

### 3. 新しい言語リソースファイルを追加

**例: `settings.json`を追加する場合**

```bash
# 1. ファイルを作成
touch apps/web/messages/en/settings.json
touch apps/web/messages/ja/settings.json
```

```json
// apps/web/messages/en/settings.json
{
  "title": "Settings",
  "profile": "Profile Settings",
  "save": "Save Changes"
}
```

```json
// apps/web/messages/ja/settings.json
{
  "title": "設定",
  "profile": "プロフィール設定",
  "save": "変更を保存"
}
```

**2. `messages.ts`に追加**
```typescript
// apps/web/lib/i18n/messages.ts
import settingsEn from '@/messages/en/settings.json';
import settingsJa from '@/messages/ja/settings.json';

const messages: Record<string, Record<string, any>> = {
  en: {
    ...commonEn,
    ...authEn,
    ...dashboardEn,
    settings: settingsEn,  // ← 追加
  },
  ja: {
    ...commonJa,
    ...authJa,
    ...dashboardJa,
    settings: settingsJa,  // ← 追加
  },
};
```

**3. コンポーネントで使用**
```tsx
const { t } = useI18n();
console.log(t('settings.title'));  // "Settings" または "設定"
```

## 言語検出フロー

1. **Cookie (`NEXT_LOCALE`)** - ユーザーが明示的に選択した言語
2. **Accept-Language ヘッダー** - ブラウザ設定
3. **デフォルト (`en`)** - フォールバック

### 言語切り替えの動作

```tsx
const { setLocale } = useI18n();

// ユーザーが日本語を選択
setLocale('ja');

// 内部動作:
// 1. Cookie `NEXT_LOCALE=ja` を設定（有効期限: 1年）
// 2. ページをリロード（言語リソースを再読み込み）
// 3. 次回アクセス時も日本語で表示
```

## ベストプラクティス

### ✅ 推奨

```tsx
// 1. ネストされたキー構造を使用
t('sessions.new.scenario.title')  // ✅ 明確で管理しやすい

// 2. パラメータ化を活用
t('dashboard.welcome', { name: user.name })  // ✅ 動的なテキスト

// 3. 英語をデフォルトに
// 翻訳が見つからない場合は英語にフォールバック
```

### ❌ 避けるべき

```tsx
// 1. ハードコードされた文字列
<h1>Dashboard</h1>  // ❌ 翻訳不可能

// 2. フラットなキー構造
t('sessions_new_scenario_title')  // ❌ 管理が困難

// 3. 翻訳キーの直接表示
{t('missing.key')}  // ❌ キー名が表示される（開発時は問題ないが、本番では注意）
```

## トラブルシューティング

### 問題: 翻訳が見つからない

**症状:**
```tsx
t('dashboard.new_key')  // → "dashboard.new_key" がそのまま表示される
```

**解決方法:**
1. キーが正しいか確認: `apps/web/messages/en/dashboard.json`
2. `messages.ts`にインポートされているか確認
3. 開発サーバーを再起動: `npm run dev`
4. ブラウザのコンソールで警告を確認: `Translation missing: dashboard.new_key (locale: en)`

### 問題: `useI18n must be used within I18nProvider`

**症状:**
```
Error: useI18n must be used within I18nProvider
```

**原因:**
- `useI18n`を`I18nProvider`の外で使用している
- `'use client'`ディレクティブが欠けている

**解決方法:**
```tsx
// ❌ 間違い
export default function Page() {
  const { t } = useI18n();  // Server Componentではエラー
}

// ✅ 正しい
'use client';  // ← 追加

export default function Page() {
  const { t } = useI18n();  // OK
}
```

### 問題: 言語切り替えが動作しない

**症状:**
- `setLocale('ja')`を呼び出しても言語が変わらない

**解決方法:**
1. Cookieが正しく設定されているか確認: DevTools > Application > Cookies > `NEXT_LOCALE`
2. ページがリロードされているか確認（`setLocale`は自動的にリロードします）
3. ブラウザキャッシュをクリア

## next-intl との比較

| 項目 | カスタム useI18n | next-intl |
|------|-----------------|-----------|
| **軽量性** | ✅ 非常に軽量（~100行） | ❌ 依存関係が重い |
| **学習コスト** | ✅ シンプル | ⚠️ ドキュメント読解必要 |
| **柔軟性** | ✅ 完全にカスタマイズ可能 | ⚠️ ライブラリの制約あり |
| **機能** | ⚠️ 基本機能のみ | ✅ 豊富（複数形、日付、数値フォーマット） |
| **型安全性** | ⚠️ 手動で型定義 | ✅ 自動型生成 |
| **メンテナンス** | ❌ 自社で対応 | ✅ コミュニティ |

**結論:**
- **現状**: カスタム`useI18n`で十分（基本的な翻訳のみ必要）
- **将来**: 複数形、日付フォーマット等が必要になれば`next-intl`への移行を検討
